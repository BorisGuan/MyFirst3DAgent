Tool family mapping:
- add_panel_lines, add_parting_lines, and refine_surface use surface_detailing.
- add_armor_layers and add_vents use armor_composition.
- add_pipes, add_hydraulic_rods, and add_weapon_mounts use mechanical_attachment.
- add_thrusters uses propulsion_detailing.
- add_sensors uses sensor_detailing.
- add_surface_damage with surface_damage or weathering uses damage_weathering.

Recommended method mapping:
- add_panel_lines -> engraved_line_layout.
- add_parting_lines -> parting_line_planning.
- add_armor_layers -> layered_plate_blockout.
- add_vents -> recessed_vent_layout.
- add_thrusters -> thruster_component_placement.
- add_pipes -> curve_pipe_routing.
- add_hydraulic_rods -> piston_pair_layout.
- add_sensors -> sensor_module_placement.
- add_weapon_mounts -> hardpoint_mount_layout.
- add_surface_damage with surface_damage -> shallow_damage_marking.
- add_surface_damage with weathering -> weathering_pass_planning.
- refine_surface -> surface_cleanup_pass.

Automation guidance:
- Use manual_guided when the plan can only become human instructions now.
- Use semi_automatable when future mesh access could assist with paths, markers, or simple placement.
- Use automation_candidate when the operation is a good V0.4 candidate after mesh access exists.
- Use blocked_until_mesh when real surface boundaries, connection points, or thickness are essential.
- Use not_recommended when automation would likely damage structural logic or produce misleading output.

Difficulty guidance:
- low: mostly module placement, label generation, or simple orientation hints.
- medium: needs paths, surface regions, paired points, or zone interpretation.
- high: needs real mesh, booleans, topology, thickness, fit, collision, or assembly logic.
- unknown: insufficient information.

Risk guidance:
- low: ordinary design caution only.
- medium: may affect readability, print clarity, fitting, or local design coherence.
- high: may weaken structure, interfere with articulation, or needs real mesh data before execution.
- unknown: insufficient risk information.

Zone mapping guidance:
- Use placement_zones from OperationPlan V2 as input_zone values.
- If placement_zones is empty, return an empty zone_mapping array.
- Map broad visible armor surfaces to primary_visible_surface.
- Map edges, borders, and outer rims to edge_detail_zone.
- Map joint_surrounds, lower_chest_edge, root_connection, and moving areas to joint_sensitive_zone.
- Map hardpoints, backs, side mounts, and equipment connections to mounting_zone.
- Map battle damage or weathering zones to damage_zone.
- Map sensors, cameras, head optics, and antenna orientation areas to sensor_orientation_zone.
- Use unknown_zone only when the zone cannot be interpreted conservatively.

Rules:
- Preserve target_part exactly from OperationPlan V2.
- Do not add new OperationPlan V2 fields.
- Do not output executable Blender, Python, CAD, or mesh commands.
- Do not claim that geometry has been generated.
- blocked_by should use short labels such as no_real_mesh, no_surface_boundaries, no_part_connection_points, no_joint_range_data, no_thickness_data, no_mounting_surface, no_scale_parameters.
- risk_reasons should be based on risk_notes first, then operation context.
- mitigation_steps should be practical and conservative.
- execution_steps should be useful to a human modeler or future execution layer, but must not be software-specific commands.
- execution_brief should clearly state what to do first and what is blocked by missing real model data.
- Return JSON only. Do not include markdown or extra explanation.
